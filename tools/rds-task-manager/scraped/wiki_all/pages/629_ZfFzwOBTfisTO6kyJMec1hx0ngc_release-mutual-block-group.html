Release Mutual Block Group​

Modified May 16, 2024
Interface introduction​
This interface is used when SEER Robotics vehicle and third-party vehicle need to interact to achieve that only one vehicle can be in working state at the same time in the same area. Detailed mutual block resource occupation policy see resource occupation logic.​
Request​
•
Function: Release the specified mutual block group.​
•
Method: post​
•
Interface Description: /releaseBlockGroup​
​
Code block​
POST "http://host:8088/releaseBlockGroup"​
​
Parameter introduction​
​
Name​
Type​
Description​
Required​
id​
string​
Occupant id, the occupant id cannot be the robot name in RDSCore.​
Yes​
blockGroup​
string array​
Mutual block group name array, mutual block group name can be configured in roboshop, if the blockGroup is empty, all mutual block groups are released.​
Yes​
​
Request example​
​
Code block​
{​
"id": "car1",​
"blockGroup": ["BlockGroup1","BlockGroup2"]​
}​
​

​
Name​
Type​
Description​
blockGroup​
string​
Mutual block group name​
status​
bool​
Release status, true = release successful, false = release failed.​
