Paging Query Action Block Information​

Modified May 17, 2024
Request​
•
Function: Query the action blocks in the system through parameter paging. The page and size in the query parameters are examples, which means this interface can filter and sort in multiple ways.​
•
Method: Get​
•
Interface Description: /blocks​
​
Code block​
GET "http://host:8088/blocks"​
​
Request parameters​
​
Name​
Type​
Description​
Required​
page​
number​
The number of pages, if not passed, defaults to 1.​
No​
size​
number​
The number of action blocks per page, if not passed, defaults to 20.​
No​
orderBy​
string​
Sorting field​
No​
orderMethod​
string​
Sort by action block update time. Can be descending and ascending, default is descending.​
No​
where​
object string​
Filter, JSON string, do not filter when not passed, see ​Paging Query Action Block Information.​
No​
​
​
Filter​
​
Code block​
{​
"relation":"AND", // Represents the combination of multiple query conditions, supporting AND, OR​
"predicates":[ // The conditions for each column are in the format of [negation (optional), column name, operator, value], three strings, and the supported operations are: GT, LT, EQ, Iike, NE, IN.​
["location", "EQ", "LM7"], // location is LM7​
["NOT", "operation", "EQ", "JackLoad"], // operation is not JackLoad​
["state", "EQ", "FINISHED"] // state is FINISHED​
]​
}​
